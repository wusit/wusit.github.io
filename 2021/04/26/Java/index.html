Java从入门到放弃

注意事项

  • 源代码文件名必须与公共类的命名相同，严格区分大小写
  • 运行已编译程序时，从类的main方法开始执行，所以类的源文件中必须包含一个main方法
  • 函数调用不用->, 而是.
  • JAVA整型范围与运行代码的机器无关；可以为数字加下划线，比如1_000_000， 增加可读性，编译器会自动去除
  • 不存在任何无符号的整数类型
  • 可以在一行声明多个变量比如int i, j; 不过不推荐这种写法，逐一声明可提高代码可读性
  • 声明一个变量之后，必须使用赋值语句对变量进行显示初始化，不能使用未初始化变量
  • 常量定义使用final关键字，可在类内部使用static final定义。const为保留关键字，目前未使用
  • 枚举类型使用enum Sizes { V1, V2 }，Size s = Size.V1；Size类型变量，只存储枚举中的值和null
  • 整数值和布尔值之间无法转换
  • for循环中，检测两个浮点数是否相等要格外小心，因为有计算误差，所以可能会造成死循环
  • 直接使用=进行数组变量间的赋值，两者是共享地址空间；需要用Arrays.copyOf进行空间地址分离
  • 日期类有Date和LocalDate、LocalDateTime，一般使用LocalDate、LocalDateTime
  • 当且仅当一个类没有构造器时，系统默认提供一个无参数构造器，用来类属性默认值的设置
  • 如果类中提供了至少一种构造器。但是没提供无参数构造器，那么在实例化时如果没有提供参数会报错
  • 在PHP的函数或方法内部，可以声明或创建局部变量，但在循环、条件、其他代码块中没有“局部作用域变量”的概念
  • JAVA方法体、循环体、条件分支、其他代码块内，都可声明局部作用域变量。比如
    • 循环中的i在外部无法使用
    • 循环内的变量在外部无法使用，除非是在外部进行定义
    • 可在两个不同的循环中定义同名的局部变量

数据类型

  • 整型（int、short、long、byte）
  • 浮点（float、double）
  • 字符（char）
  • 布尔（boolean）

类型转换

  • 强制类型转换通过截断小数点部分将浮点值转化为整型
  • 不要将bool类型与其他任何类型进行强制转换，可避免错误。极少数下可使用三元? 1 : 0转换

字符串

  • Substring，第一个参数是起点Index，第二个参数不是长度，而是终点Index（不包括）
  • 连接两个字符串可以使用String.join(“分隔符”, 元素1, 元素2…)
  • JAVA中的字符串，不是C++中的字符数组char greeting[] = “hello”
  • 比较两个字符串相等，用equals，两个值都可以是变量或者字面量，不要用==，这时判断地址的
  • equals对应有个equalsIgnoreCase，忽略大小写检测字符串是否相等的

类

  • 一个文件可以包括多个类：只能一个public公有类，且名字和文件名相同；其余非公有类没个数限制

  • 构造器与类名同名。可以有多个，参数个数或者类型不同即可，没有返回值

  • 访问实例域，最好使用this.param，而不是直接param，可读性更强，更好的与局部变量区别开

  • 不要编写返回引用可变对象的访问器方法，会破坏封装；如果需要返回可变对象的拷贝，就使用clone

  • 确保在每一个构造函数执行之后，属性值被设置，后面不会改变。就可以把这个属性值设置为final

  • 静态变量和PHP性质一样，都是所有的实例对象共享一个内存空间

  • 静态常量 static final， 比如System.out， out就是一个静态常量

  • 最好不要将类属性设置为public，但是static final没问题，因为final表明了是一个常量不可修改（PHP中的常量const）

  • 静态方法和PHP性质一样。不用实例化即可访问；当然也可以使用实例化对象调用静态方法（不建议，可读性差，并且方法和实例化对象还没关系）

  • PHP访问静态方法使用:: JAVA使用.

  • 每个类都可以有一个main入口方法， 如果单独测试，会执行，如果是项目中的一部分，被引用，则永远不会被执行

  • JAVA在一个类中允许重载任何一个方法，不止是构造方法

  • 类属性是有默认值的，比如int默认是0. 但是局部变量必须明确初始化（PHP默认值都一样是null），但是不建议，可读性差

  • 只能使用导入一个包，不能使用比如import java. import java.*.*导入以java为前缀的所有包

继承

  • 使用super.function调用超类方法
  • 如果子类没有主动调用超类的构造方法， 将自动调用无参数的默认构造函数、
  • JAVA不支持多继承
  • 对象变量是多态的，A a = new A(); a可以是A类的任何一个子类对象
  • 覆盖方法时，子类方法的可见性不能低于超类
  • 类声明为final时，不能被继承扩展
  • 抽象类不能被直接实例化，必须用子类来继承实现其中的所有抽象方法
  • 可以定义一个抽象类的对象变量，但是只能引用真实的子类对象
  • 如果没有明确指定要继承的超类，默认是继承自Object类
  • JAVA中除了基本类型，其余的都是对象，包括数组
  • 只要是对象与一个字符串进行+连接，就会自动调用对象的toString方法

泛型程序设计

泛型注意事项

  • 泛型类指的是实例化时，指明泛型的具体类型

  • 泛型方法指的是在调用方法时指明，指明泛型的具体类型

  • 擦拭法实现泛型

    • 编译器把类型<T>视为Object；编译器根据<T>实现安全的强制转型
    • Java的泛型是由编译器在编译时实行的，编译器内部永远把所有类型T视为Object处理
    • 在需要转型的时候，编译器会根据T的类型自动为我们实行安全地强制转型
    • 无法取得带泛型的Class： 因为T是Object，我们对Pair<String>和Pair<Integer>类型获取Class时，获取到的是同一个Class，也就是Pair类的Class; 换句话说，所有泛型实例，无论T的类型是什么，getClass()返回同一个Class实例，因为编译后它们全部都是Pair<Object>。
    • 无法判断带泛型的类型，原因同上

  • 泛型类型T不能用于静态方法

  • 泛型方法的返回值前面加上<K>

  • 如果一个方法需要的参数是Demo<Number>，真实传入的泛型类型是Demo<Integer> ，会报错，因为Demo<Integer>不是Demo<Number>的子类，主要问题在于方法类型定死了只能传入Number，这时候可以使用Demo<? extends Number>，使得方法接收所有泛型类型为Number或Number子类的Demo类型，称为上界通配符

  • 使用<? extends Number>通配符

    • 内部可以调用获取Number引用的方法
    • 内部无法调用传入Number引用的方法，null除外
    • 即只可读，不可写

  • 使用<T extends Number>通配符表示泛型类型限定为Number和Number的子类

  • 使用<? super Integer>下界通配符代表可传入Integer和父类，但是不可读，只可写

  • 当不需要使用类型的具体功能时，可以直接使用Demo<?>通配符表示未知类型

  • 继承或实现泛型类/接口时，可以明确/不明确声明泛型类型

  • 是否拥有泛型方法，与其所在的类是不是泛型类没有关系，不要和抽象类混淆
